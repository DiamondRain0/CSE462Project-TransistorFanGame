# CSE462 Game Project

![transistor](https://github.com/user-attachments/assets/730d4b02-7b25-4152-8dac-953cdf30d05a)

This repository contains a C++ project developed for the CSE462 course. The program implements a board game with an AI opponent and a simple user interface built on **SFML 2.6.1-1**. It is structured to separate interface, game logic, and AI components with a Makefile to streamline builds.

## 📁 Project Structure

```
Makefile            # build rules
Readme              # (this file)
include/            # header files
    AI.h
    Board.h
    Constants.h
    GameUI.h
    Theme.h
obj/                # compiled object files (build output)
src/                # source code
    AI.cpp
    Board.cpp
    GameUI.cpp
    main.cpp
```

## 🛠️ Build Instructions

This project depends on **SFML 2.6.1-1** for graphics and windowing. Ensure SFML is installed and its include/lib paths are accessible to the compiler.

Make sure you also have a C++ compiler installed (e.g. `g++`). From the root directory:

```sh
make              # compiles the project and places object files in `obj/`
./game.exe        # runs the executable generated by `main.cpp`
```

Alternatively, you can use the provided VS Code C/C++ build task (`Ctrl+Shift+B`) which compiles the active file.

## 🎮 Features

- Game board management (`Board` class)
- Artificial Intelligence opponent (`AI` class) using Minimax search with alpha-beta pruning
- Simple text-based UI with theming support (`GameUI`)
- Constants and configuration in `Constants.h` and `Theme.h`

## � AI Implementation Details

The AI component is located in `src/AI.cpp` and implements a depth-limited Minimax algorithm enhanced with alpha-beta pruning for efficiency. Key points include:

- **Initialization**: A fast lookup table of neighbors (`neighbors` array) is precomputed by `initAI()` to speed up move generation.
- **Evaluation Function**: `evaluate()` scores a board based on player mobility. It counts empty adjacent squares for each player and applies a heuristic `p1Mobility - 2 * p2Mobility` to make the AI aggressive.
- **Move Generation**: Instead of scanning the entire board, `getRelevantRemovals()` restricts removal candidates to the vicinity of both players. This reduces branching factor and increases speed.
- **Minimax with Alpha-Beta**: `minMax()` recursively explores moves and removals up to a fixed depth (`MAX_DEPTH`). Alpha-beta pruning cuts off branches that cannot influence the final decision, dramatically reducing search time.
  - Maximizing nodes represent AI turns; minimizing nodes represent the human player.
  - The algorithm simulates piece movement and square removal, undoing each change after evaluation.
- **Best Move Selection**: `getBestMove()` wraps everything, iterating over legal moves, scoring each with `minMax()`, and selecting the move/removal pair with the highest value.

These optimizations allow the AI to search several plies deep within a reasonable time frame in a 7x7 board game.

## �🧩 How to Use

1. Build the project using the instructions above.

- If the build fails, ensure your compiler path is correct and `make` is installed.
